# 基础版作业任务书：从声音表示到 CNN + LSTM 基线闭环

## 一、任务定位

基础版不是“最简单的部分”，而是整门课程中最关键的起步阶段。它承担的任务是：帮助你第一次真正建立“鸟鸣如何一步步变成分类结果”的完整认识。

基础版对应课程中的三个核心内容：

1. 声音为什么要先转成频谱图。
2. CNN 怎样从频谱图中提取局部特征。
3. LSTM 为什么还要继续读取时间顺序。

如果基础版完成得扎实，后续的 ResNet18 + BiLSTM 深入版和 CNN + Transformer 探索版，才会建立在清晰的结构理解之上。

## 二、适合人群

建议以下同学优先选择基础版：

1. 第一次接触音频分类任务。
2. 对 CNN 或 LSTM 有模糊印象，但没有独立跑通过完整实验。
3. 想先建立一条可运行、可解释、可复盘的最小闭环。

## 三、本版完成后你应达到的状态

完成基础版后，你至少应当能够做到：

1. 说清楚为什么原始音频通常不会直接送进分类模型，而是先变成频谱图或 Mel 频谱图。
2. 说清楚 CNN 和 LSTM 在你的模型里分别负责什么。
3. 拿出一个真正跑通的基线流程，并给出基本结果。
4. 对至少一部分正确样本和错误样本作出初步解释，而不是只报告准确率。

## 四、建议按以下六步推进

### 第 0 步：建立起点，不急着开写代码

先用一小段文字说明你当前对任务的理解，并写下你最不确定的几个问题。

建议先回答：

1. 鸟类声音识别和普通图像分类最大的不同在哪里。
2. 为什么声音可以转成图来处理。
3. 为什么 CNN 后面还要接一个时序模型。

### 第 1 步：认识数据，先建立样本台账

不要一开始就把数据当成“现成输入”。先了解有哪些鸟种、每类样本是否均衡、音频大致有什么质量差异。

建议重点观察：

1. 每个鸟种的音频数量是否接近。
2. 音频时长是否差异很大。
3. 是否存在明显噪声、静音、环境声过强等情况。
4. 同一种鸟在不同样本中的叫声节奏是否完全一致。

### 第 2 步：完成“声音变图”的理解与记录

这一阶段的目标不是炫耀参数，而是把“声音如何被表示”说清楚。你需要让自己明白，也让别人看得懂：为什么这一步是后续模型成立的前提。

建议围绕以下问题组织说明：

1. 原始波形为什么不利于直接做分类判断。
2. 频谱图如何同时体现时间、频率和能量信息。
3. 为什么 Mel 频谱图更适合本课程中的鸟类音频任务。

### 第 3 步：搭建一个最小可解释的 CNN + LSTM 模型

本步的核心不是追求复杂，而是构建一个结构清楚、输入输出明确、能作为后续比较起点的基线模型。

你至少应当说清楚三件事：

1. 模型输入是什么。
2. CNN 在看什么，LSTM 又在继续读什么。
3. 模型为什么可能对这个任务有效。

### 第 4 步：完成第一次完整训练与结果观察

这一阶段的重点是获得第一轮真实结果，并把“训练过程发生了什么”记录下来，而不是只截一张最终分数图。

建议重点记录：

1. 训练过程中 loss 和准确率的大致变化趋势。
2. 是否出现明显异常，例如训练不稳定、过早停滞、验证集波动过大。
3. 第一轮结果是否符合你在第 3 步中的预期。

### 第 5 步：做第一次像样的错误分析

请从“结果展示”走向“结果解释”。至少挑出一部分正确样本和错误样本，分析模型可能为什么判断成这样。

建议分析的角度：

1. 两个鸟种是否在频谱图上存在相似纹理。
2. 错误样本是否带有明显背景噪声或无效片段。
3. 某些样本是否时长不足、叫声不完整，导致信息不够充分。
4. 模型是否更依赖某些明显节奏模式。

## 五、本层建议至少保留的产出

1. 任务理解与问题清单。
2. 样本统计与数据风险判断。
3. 输入表示说明与示意图。
4. CNN + LSTM 模型说明、训练曲线与第一轮结果摘要。
5. 至少两组错误样本分析。

## 六、LLM 可以在哪些地方帮你，但不能替你完成什么

### 可以使用的方向

1. 帮你解释概念，例如频谱图、卷积特征、LSTM 的作用。
2. 帮你梳理实验记录结构。
3. 帮你分析某类错误现象可能有哪些原因。
4. 帮你把零散观察整理成更清楚的表达。

### 不建议的使用方式

1. 直接让 LLM 代写整份报告，然后自己不做核对。
2. 直接复制模型解释，却不知道这些话对应自己的实验哪个部分。
3. 把每个报错都当成“求现成修复答案”，而不记录自己的排查过程。

## 七、基础版完成标志

当你满足以下状态时，可以认为基础版已经完成：

1. 你拿到了一个完整可运行的 CNN + LSTM 基线结果。
2. 你能用自己的话解释“声音变图”“CNN 提特征”和“LSTM 读顺序”这三件事。
3. 你留下了数据说明、训练过程、结果摘要和错误分析。
4. 你已经能清楚指出：如果继续做深入版，最值得改进的地方是什么。
