# 课程作业总说明：从鸟鸣到结构解释

## 一、作业定位

本课程作业不再采用“一份要求覆盖所有同学”的写法，而是拆分为三个难度层级，分别对应平台中的三套实验路径：

1. 基础版：CNN + LSTM
2. 深入版：ResNet18 + BiLSTM
3. 探索版：CNN + Transformer

这样拆分的目的，不是把同学简单分成高低水平，而是让每个人都能在当前基础上完成一条清晰、可落地、可复盘的学习路线。

本次作业的核心不只是“训练出一个模型”，而是完成以下三个层面的转变：

1. 从“听见鸟鸣”走向“理解声音如何被表示”。
2. 从“会跑一个模型”走向“能解释为什么这样设计”。
3. 从“看到一个结果”走向“能分析结果为什么会这样”。

## 二、三档路径与课程内容的对应关系

| 路径 | 对应平台描述 | 对应课程内容重点 | 适合人群 |
|------|--------------|------------------|----------|
| 基础版 | 先把全流程跑通 | 声音变图、CNN + LSTM、最小闭环 | 第一次接触音频分类或时序模型的同学 |
| 深入版 | 进入结构升级与比较 | ResNet18 + BiLSTM、结构比较、结果解释 | 已能完成基线模型，希望做有效比较的同学 |
| 探索版 | 把实验推进成小研究 | CNN + Transformer、注意力机制、误差分析 | 已完成前两层，希望继续探索结构差异的同学 |

这三档路径同时对应课程中的知识递进：

1. 基础版重点承接“声音如何变成频谱图”以及“CNN 和 LSTM 如何分工”。
2. 深入版重点承接“为什么要升级卷积主干，以及为什么需要双向上下文”。
3. 探索版重点承接“当 LSTM 系列已经比较清楚后，Transformer 是否值得引入”。

## 三、请按以下方式使用这些文件

1. 先阅读本总说明，确认自己当前适合从哪一档开始。
2. 再进入对应的独立任务书，按其中的分步任务推进，而不是只盯着最后的准确率。
3. 如果你想直接做深入版或探索版，也可以，但必须补齐前一层中要求的关键过程材料，否则后续分析会失去依据。
4. 若中途发现任务难度判断失准，可以下调或上调，但请在实验记录中写明调整原因。

## 四、三个独立任务文件

本目录下现已拆分为以下三个文件：

1. 基础版作业任务书：从声音表示到 CNN + LSTM 基线闭环。
2. 深入版作业任务书：从 CNN + LSTM 走到 ResNet18 + BiLSTM。
3. 探索版作业任务书：从 ResNet18 + BiLSTM 走向 CNN + Transformer。

## 五、所有版本共同要求

无论选择哪一档，都建议保留以下四类过程材料。

### 1. 问题清单

请在作业开始前先写下你当前最不确定的 3 到 5 个问题，例如：

1. 为什么音频分类通常先转成 Mel 频谱图。
2. 为什么 CNN 后面还要接时序模型。
3. 如果模型效果不好，应先检查数据、输入表示、模型还是评估方式。

### 2. 过程记录

请尽量按阶段记录，而不是等全部完成后再回忆补写。至少应覆盖：

1. 你最初准备怎么做。
2. 中途遇到了什么困难。
3. 你如何修改原方案。
4. 哪些尝试没有成功，以及你从中得到了什么判断。

### 3. 关键结果材料

建议至少保留以下结果：

1. 数据划分或样本组织说明。
2. 训练过程中的核心曲线或关键日志。
3. 至少一组正确样本与错误样本分析。
4. 最终结论与尚未解决的问题。

### 4. LLM 辅助学习记录

本课程鼓励使用大语言模型，但要求把它作为学习伙伴，而不是代做工具。建议重点保留以下类型的对话：

1. 你原本不理解的概念是怎样被解释清楚的。
2. 你遇到报错或异常现象时，LLM 给了哪些排查路径。
3. 你如何判断某条建议值得采用，或为什么决定不采用。
4. 你怎样把 AI 的回答转写成自己的理解表达。

## 六、评价关注点

课程作业更看重“任务推进质量”和“思考深度”，而不是单一结果数字。评价时主要关注：

1. 你是否真的理解了自己在做什么。
2. 你是否能把课程中的概念与自己的任务步骤对应起来。
3. 你是否进行了有效记录，而不是只提交最终结果。
4. 你是否能解释模型成功或失败背后的原因。
5. 你是否合理使用了 LLM，并保留了自己的判断过程。

## 七、建议的完成顺序

建议优先采用“逐级解锁”的方式：

1. 先在基础版中建立 CNN + LSTM 的完整流程与结果意识。
2. 再在深入版中比较 ResNet18 + BiLSTM 相比基础版到底改变了什么。
3. 最后在探索版中进入 CNN + Transformer，并把比较组织成更像小研究的表达。
