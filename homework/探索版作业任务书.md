# 探索版作业任务书：从 ResNet18 + BiLSTM 走向 CNN + Transformer

## 一、任务定位

探索版不是“把模型换得越新越好”，而是把课程作业推进到研究型表达层面。你需要从“完成一个规定动作”转向“围绕一个明确问题设计实验，并给出有依据的结论”。

探索版对应课程知识链的最后一层：当你已经理解声音表示、CNN、LSTM、BiLSTM 这条主线之后，下一步要学习的，是如何把 CNN + Transformer 放进一个明确的问题中，继续提出比较、分析误差，并把实验变成可讨论的结论。

## 二、建议前提

建议满足以下条件后再进入探索版：

1. 你已经完成基础版和深入版，或具备同等质量的基线与比较结果。
2. 你已经能够用自己的话解释当前 ResNet18 + BiLSTM 模型为什么成立。
3. 你不只是想“再试一个模型”，而是确实有一个想验证的问题。

## 三、本版真正要做的事

探索版最重要的不是更复杂，而是更清楚。你需要完成以下三件事：

1. 选定一个值得探索的问题。
2. 围绕这个问题设计一组尽量公平的比较。
3. 对结果做出有证据的解释，而不是只报“谁更高”。

## 四、建议优先选择一个主问题，不要同时开太多线

你可以从以下方向中选择一个作为主要探索轴：

1. 结构探索：例如 CNN + Transformer 相比 ResNet18 + BiLSTM 是否对本任务有帮助。
2. 数据探索：例如加噪、时移、变调、SpecAugment 是否提升鲁棒性。
3. 评估探索：例如 K 折交叉验证是否改变你对模型优劣的判断。
4. 错误分析探索：例如哪些鸟种最容易混淆，混淆究竟来自局部纹理、顺序信息，还是全局依赖。
5. 复杂度探索：例如更复杂的模型是否真的值得对应的训练成本和调参成本。

## 五、建议按以下七步推进

### 第 0 步：提出一个像样的问题，而不是泛泛地说“我想优化模型”

请把探索目标写成一个尽量具体的问题。例如：

1. 在当前鸟类数据上，当 ResNet18 + BiLSTM 已能提取较稳的时频特征后，引入 Transformer 是否还能进一步缓解某些仍然容易混淆的类别。
2. Transformer 带来的收益，到底更多体现为总体提升，还是体现在少数具有长程依赖的样本上。
3. 更复杂的注意力结构是否只是让模型更重，却没有带来足够明确的增益。

### 第 1 步：回顾已有结果，确定探索不是盲试

把基础版和深入版的已有结论重新整理出来，找出真正值得追问的地方。探索版不是凭兴趣乱试，而是从已有现象中长出来的。

建议回顾：

1. 哪些类别在 ResNet18 + BiLSTM 下仍然表现较差。
2. 哪些错误在深入版中仍然存在。
3. 哪些改进看起来有效，但证据其实并不充分。

### 第 2 步：设计实验矩阵，先控制范围再开始尝试

把你准备比较的方案列清楚，避免做到一半才发现变量过多、难以解释。

建议至少明确：

1. 基准模型是谁，例如 CNN + LSTM 或 ResNet18 + BiLSTM。
2. 你准备新增或调整的因素是什么。
3. 你打算保留哪些不变条件，以保证比较仍然有意义。
4. 你将以什么结果作为主要判断依据。

### 第 3 步：执行探索，但每一次尝试都要带着记录

这一阶段不只是跑实验，而是保留“为什么这么改”和“改完发生了什么”。

建议重点记录：

1. 每次尝试对应的目标是什么。
2. 结果相比基准出现了什么变化。
3. 变化是否稳定，还是只在个别设置下偶然出现。
4. 如果结果不如预期，最可能的问题出在哪里。

### 第 4 步：把评估做得更有说服力

探索版不应只看单一准确率。你需要尽量从多个维度判断改动是否真的有效。

建议考虑：

1. 总体表现是否提升。
2. 结果是否稳定，而不是只在一次划分上碰巧较好。
3. 提升是否集中在某些特定类别。
4. 改动是否只是让模型更复杂，却没有带来实质收益。

### 第 5 步：做更深入的错误分析，而不是只看好结果

请专门挑选那些最能体现问题本质的错误样本，做有针对性的分析。

建议聚焦：

1. 是否存在一对长期容易混淆的鸟种。
2. 这类混淆更像是局部纹理相似、节奏相似，还是全局依赖没有建模好。
3. 新方法解决了哪些错误，又引入了哪些新问题。
4. 如果继续做下一轮改进，你最想优先动哪里。

### 第 6 步：把结果转化成结论，不要停留在现象罗列

这一阶段的重点是回答你的原始问题，而不是简单重复实验结果。

建议用以下方式组织结论：

1. 我的原始问题是什么。
2. 我的假设是否得到支持。
3. 支持或不支持的证据是什么。
4. 哪些部分仍然不能确定。
5. 这项探索对理解鸟类音频任务带来了什么新的认识。

## 六、建议至少保留的产出

1. 明确研究问题与初步假设。
2. 进入 Transformer 的理由说明。
3. 多组结果对照与可信度判断。
4. 典型错误案例和错误链分析。
5. 正式结论、局限与后续改进建议。

## 七、LLM 在探索版中的正确位置

### 它最适合帮你的地方

1. 帮你把一个模糊想法收束成可研究的问题。
2. 帮你审视实验矩阵是否合理。
3. 帮你从结果中提炼可能的解释路径。
4. 帮你把“现象描述”进一步整理成“结论表达”。

### 你仍然必须自己完成的部分

1. 研究问题的最终确定。
2. 结果是否可信的判断。
3. 哪些证据足以支撑结论，哪些仍只是猜测。
4. 你对失败尝试的取舍和解释。

## 八、探索版完成标志

当你达到以下状态时，可以认为探索版已经完成：

1. 你提出了一个明确且有依据的问题。
2. 你围绕这个问题完成了相对清晰的比较设计。
3. 你不只给出了结果，还给出了证据、解释与局限。
4. 你的作业已经不再只是“完成任务”，而开始具备“小型研究”的表达质量。
